# Refactoring del layout condiviso dell'applicazione

## Centralizzazione della struttura base e degli elementi comuni

### ✨ Nuove Funzionalità (Added)

- Introdotto componente `Layout.astro` configurabile tramite props:
  - `title`
  - `eyebrow`
  - `subtitle`
  - `backLink`
  - `backLabel`
- Aggiunto supporto opzionale per link di navigazione "back" nell'header.
- Aggiunto footer condiviso con branding "Ruota di Verona · Study Hub".

### 🔄 Modifiche (Changed)

- Sostituito il layout Astro di default con una struttura applicativa personalizzata.
- Centralizzati nel layout comune:
  - metadati HTML principali;
  - tipografia (`Nunito`);
  - variabili CSS globali;
  - header, contenitore principale e footer.
- La homepage ora utilizza il componente `Layout` invece di definire localmente l'intera struttura HTML (`html`, `head`, `body`, `header`, `main`).
- Ridotta la duplicazione di markup e stili tra le pagine.

### 🎨 Miglioramenti UI

- Uniformata la presentazione visiva dell'applicazione attraverso un layout condiviso.
- Standardizzati header, spaziature, colori e tipografia.
- Migliorata la consistenza grafica per future pagine e sezioni.

### Risultato finale

La struttura dell'applicazione è stata consolidata in un layout riutilizzabile che centralizza stile e componenti comuni, semplificando la manutenzione e garantendo un'interfaccia coerente tra le diverse pagine del progetto.
